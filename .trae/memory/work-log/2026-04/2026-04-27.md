5. 实现设备展示内容在页面中间垂直居中，并设置上下 20rpx 的边距。
6. 为“添加设备”按钮绑定 `addDevice` 点击事件。

8. 解决因 `u-tabbar` 默认占位符导致的居中内容偏上问题。
9. 修复设备列表中文字内容被挤压、无法正常上下排列的问题。
10. 在设备列表项中添加了电量显示模块。
11. 修复设备列表项（device-item）中设备信息无法上下占据满空间（`justify-between` 失效）的问题。

### 技术要点
- 使用 `flex-column` 和 `justify-center` 实现内容垂直居中。
- 通过 `margin-tb-20` 自定义类名控制上下边距。
- 使用 `flex-grow-1` 让内容容器占据中间剩余空间。
- 根据 `CustomTabBar.vue` 的样式（height: 132rpx, bottom: 44rpx）计算并设置 `padding-bottom: 176rpx`。
- 为 `.page-wrap` 增加相对定位，并通过 `::v-deep .tab-bar` 设置绝对定位且高度为 0，将 TabBar 从 Flex 布局流中抽离，避免其自带的 placeholder 将页面内容过度上挤。
- 使用 `flex-column` (Flexbox 垂直布局) 管理页面整体结构。
- 在 `style` 标签中使用 `page` 选择器设置根容器高度。
- 对设备列表卡片内部采用 Flex 布局（`flex flex-direction flex-grow-1`），结合固定高度（`height: 162rpx`）与精确内边距（`padding: 34rpx 0; box-sizing: border-box;`）以及 `justify-content: space-between`，使文本区域距离上下各保持 34rpx 间距，均匀分布文本元素，防止挤压。
- 将电量显示模块改为绝对定位（`top: 20rpx; right: 20rpx;`），使其独立于右侧文本布局流。
- 为左侧图片容器设置 `flex-shrink-zero` 防止被右侧文本内容挤压。
- 为当前位置文本应用了 `text-hidden` 单行溢出省略号效果。

### 重要经验
- 在 uni-app 中，若要使容器高度充满全屏，需要显式设置 `page { height: 100%; }`。
- 移除固定像素（或 rpx）的硬编码宽高，有利于不同屏幕尺寸的适配。
- 在使用 Flex 布局横向排版图片和文字时，必须为图片容器设置明确宽高以及 `flex-shrink: 0;`，以防止图片被较长文字挤压变形。
- 当父元素使用 `align-center`（垂直居中对齐）时，其内部子元素在交叉轴（垂直方向）的高度会收缩为内容本身的高度，此时子元素即使使用 `justify-between` 也无法生效。需要将父元素的 `align-center` 改为 `align-stretch` 或移除（默认即 stretch），使子元素拉伸并获得足够的高度空间。

## 功能模块：TabBar 导航栏适配

### 完成内容
1. 修复微信小程序下悬浮 TabBar 紧贴底部的问题，增加安全距离适配。
2. 修复 `calc(44rpx + env(...))` 在微信小程序部分环境下不生效导致 `bottom` 被忽略的问题。
3. 增加 JS 动态兜底：通过 `uni.getSystemInfoSync()` 计算 `safeAreaInsets.bottom`，并将偏移量注入 CSS 变量，使用 `transform` 精准抬升 TabBar。
4. 定位到 `pages/tab/device/index.vue` 中 `::v-deep .tab-bar` 覆盖样式（`position:absolute; height:0`）导致定位方案持续失效，并移除该覆盖。
5. 将 `CustomTabBar` 重构为纯自定义容器布局，不再依赖 `up-tabbar` 内部定位实现，统一使用固定定位 + 动态底部偏移，提升跨端一致性。

### 技术要点
- 在 `<up-tabbar>` 组件中设置 `:safeAreaInsetBottom="false"` 禁用默认的底部安全区占位。
- 采用“分离式偏移”方案：`bottom` 仅使用 `env/constant(safe-area-inset-bottom)` 处理安全区，`margin-bottom: 44rpx` 处理业务视觉间距，避免混合单位 `calc` 在小程序端失效。
- 采用更稳定的位移适配：`transform: translate3d(0, -offsetPx, 0)`，偏移量为 `safeAreaBottom + uni.upx2px(44)`，规避组件内联 `bottom` 和小程序环境解析差异。
- 排查样式失效时需优先检查父页面 `::v-deep` 是否覆盖子组件核心定位属性。
